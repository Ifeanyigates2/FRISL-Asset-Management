# FRISL Enterprise Asset Management System (EAMS)

## Overview
ASP.NET Core 8 web application implementing a full-featured Enterprise Asset Management System for First Registrars & Investor Services Ltd. Combines ASP.NET Core MVC for workflow screens with Blazor Server for the interactive admin dashboard.

**Branding**: Navy `#1d3461`, Gold `#c4970a`

## Architecture
- **Framework**: ASP.NET Core 8 MVC + Blazor Server
- **Database**: SQLite via Entity Framework Core 8 (file: `FrislEams.Web/eams.db`)
- **Port**: 5000 (HTTP, 0.0.0.0)
- **Runtime**: .NET 8

## Key Technical Notes
- **SQLite decimal limitation**: Never use `SumAsync()` on decimal columns. Use `.Select().ToListAsync()` then `.Sum()` in memory.
- **Tag code format**: `FRISL-YEAR-CAT-DEPT-SEQ` (auto-generated by `TagCodeGenerator`)
- **AgeBand strings** (Aging report): `"0-3 Years"`, `"3-5 Years"`, `"5-10 Years"`, `"10+ Years"` ‚Äî must match view and service
- **Storage location code**: `"STORE"` (not `"STORAGE"`)
- **Seed data**: 16 assets, 8 staff, 8 departments, 3 contractors, assignments, repairs, loans, requests, RFID events, audit session

## Running the Application
Workflow: `cd FrislEams.Web && DOTNET_CLI_TELEMETRY_OPTOUT=1 dotnet run`

The app starts on port 5000. SQLite DB is auto-created and seeded on first run. Delete `eams.db` to reset seed data.

## Deployment
- Target: autoscale
- Build: `cd FrislEams.Web && dotnet publish -c Release -o out`
- Run: `./FrislEams.Web/out/FrislEams.Web --urls http://0.0.0.0:5000`
